The first step in any Python journey is establishing a rock-solid local development environment. 
While there are countless online compilers and cloud environments available, relying on them too early robs you of the crucial experience of facing and fixing real-world setup errors. 
Programmer's primary job isn't just writing code—it's debugging.

Installing Python
Unlike JavaScript (which has a million ways to run and behave differently across environments), Python is wonderfully straightforward. It behaves the exact same way regardless of where you run it.
- Download: Go directly to python.org. The website will auto-detect your operating system (Windows, Mac, or Linux) and offer the correct installer.
- Windows Users (CRITICAL STEP): When the installer launches, look at the very bottom of the first window. You must check the box that says "Add python.exe to PATH". If you forget this, your terminal will not recognize Python commands, and you will have to reinstall it.
- Verification: Open your terminal (or Command Prompt) and type python3 --version (Mac/Linux) or python --version (Windows). If it returns a version number (like Python 3.12.1), you are successfully installed!


Why Not Use Online Compilers?
Experts strongly advises against using online environments as your primary learning tool when starting out.

Why Not Online GDB or Replit?
While platforms like Online GDB are great for instantly testing a single script, they hide the terminal, folder structures, and the real-world friction of file management. If you don't face local environment errors now, you won't know how to fix them when building real apps.

Why Not Google Colab or Anaconda (Jupyter Notebooks)?
These are incredible tools, but they are highly specialized for Data Science, AI, and Machine Learning. They allow you to write text and code in alternating "cells," making it easy to visualize data instantly.
Why Not for Beginners? If your goal is to build a web backend (Django) or an API, you will never use these notebook environments. You will use a standard code editor. Sticking to a standard editor ensures you are learning the core language, not the quirks of a specialized notebook.

Setting Up VS Code
Visual Studio Code (VS Code) is the industry standard.
- The Theme: Hitesh uses a custom theme he built called "Chai Theme". You can find it in the extensions tab!
- The Python Extension: The moment you create your first file ending in .py, VS Code will prompt you to install the official Python extension. Say Yes. This gives you syntax highlighting, auto-complete (IntelliSense), and error checking.
- Naming Conventions: While you can name a file helloChai.py (Camel Case), the Python community strongly prefers Snake Case with underscores: hello_chai.py. When in Rome, code like the Romans!

Running Code: Open the integrated terminal in VS Code and type:
- Mac/Linux: python3 hello_chai.py
- Windows: python hello_chai.py
It prints the text perfectly. No magic, just direct execution.
